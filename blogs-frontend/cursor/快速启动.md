# 快速启动指南

## 当前问题
PM2 中没有找到 `gdblogs-frontend` 进程，需要重新启动。

## 解决步骤

### 1. 检查 PM2 进程列表
```bash
pm2 list
```

### 2. 如果进程不存在，重新启动服务

**方法一：使用 PM2 启动**
```bash
cd /www/wwwroot/gdblogs/blogs-frontend
pm2 start npm --name gdblogs-frontend -- start
pm2 save
```

**方法二：如果方法一不行，尝试**
```bash
cd /www/wwwroot/gdblogs/blogs-frontend
pm2 start "npm run start" --name gdblogs-frontend
pm2 save
```

### 3. 检查服务状态
```bash
pm2 list
pm2 logs gdblogs-frontend --lines 30
```

### 4. 验证服务是否正常运行
```bash
# 检查端口是否被占用
netstat -tuln | grep 3000
# 或
ss -tuln | grep 3000

# 测试本地访问
curl http://localhost:3000 | grep -o "你好，我是" | head -1
```

## 如果 PM2 启动失败

### 检查构建是否完成
```bash
# 确认 .next 目录存在
ls -lh .next/

# 确认构建ID存在
cat .next/BUILD_ID

# 如果不存在，需要重新构建
npm run build
```

### 直接运行测试（不使用 PM2）
```bash
cd /www/wwwroot/gdblogs/blogs-frontend
npm run start
```

如果直接运行可以访问，说明代码没问题，只是 PM2 配置问题。

## 常见问题

### Q: PM2 启动后立即退出？
**A:** 检查：
1. 构建是否完成（`.next` 目录是否存在）
2. 端口是否被占用
3. 查看日志：`pm2 logs gdblogs-frontend`

### Q: 如何查看所有 PM2 进程？
**A:** 
```bash
pm2 list
pm2 status
```

### Q: 如何删除旧的 PM2 进程？
**A:**
```bash
pm2 delete gdblogs-frontend
# 然后重新启动
pm2 start npm --name gdblogs-frontend -- start
```
